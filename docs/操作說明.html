WBS 系統 — 傻瓜操作說明

把政府契約 PDF 丟進去,自動變成可以點、可以查頁碼的 WBS 工作分解表。
以下每個動作都是:打開 PowerShell(在 D:\wbs 資料夾按住 Shift 點右鍵 → 「在此處開啟 PowerShell」),貼指令,按 Enter。

1. 把契約變成 WBS(最常用)

1
把契約 PDF 放到電腦上任何地方
例如 D:\契約\某某案.pdf
2
貼這行,把路徑換成你的 PDF 
.\.venv\Scripts\wbs.exe auto D:\契約\某某案.pdf
跑多久?快速模式(mock)約 1 分鐘;真模型白天要 2–4 小時、清晨約 30 分鐘。
3
跑完後,打開結果資料夾裡的 HTML
位置:data\projects\(案名)\exports\,雙擊 wbs_v1_○○○.html 就會用瀏覽器打開互動的 WBS 圖。
💡 快速測試 vs 正式跑:設定檔 wbs.local.toml 裡 mock = false 是真模型(慢但真);想快速確認流程,暫時改成 mock = true(1 分鐘,但 WBS 是照章節排的簡易版)。

2. 打開結果會看到什麼

檔案是什麼
wbs_v1_○○○.html先開這個。WBS 樹狀圖+明細清單,可以按類別過濾(付款、驗收、維運…),點頁碼直接跳到 PDF 那一頁
wbs_v1_○○○.xlsxExcel 版,要交報告用這個
coverage.txt這份 PDF 有幾頁正常、幾頁掃描、幾頁亂碼
掃描頁和亂碼頁系統會自動跳過,不會亂編內容——這是正常行為,不是壞掉。

3. 系統切錯文件了?教它一次,以後就會了

1
先看系統自己承認哪裡沒把握 
.\.venv\Scripts\wbs.exe review list (案名)
2
叫系統產生一張「訂正卷」(Excel) 
.\.venv\Scripts\wbs.exe review annotate (案名)
會產生 data\projects\(案名)\review\annotation_L1.xlsx
3
打開那個 Excel,把錯的改對
每一列是一份子文件(起頁/迄頁/類型/標題)。類型欄有下拉選單,選就好,不用打字。改完存檔。
4
交卷 
.\.venv\Scripts\wbs.exe review accept (案名)
你的訂正會存進案例庫,變成以後的考題和修規則的依據。
「(案名)」= data\projects\ 底下的資料夾名稱,例如 contract_11108_2026-07-22。

4. 改了規則之後,怎麼確認沒有改壞?

解析規則放在 profiles\default.toml(是設定檔,不是程式)。只要動過它,就跑這行考試:

.\.venv\Scripts\wbs.exe eval run

看最後一行:

結果意思
merge_gate: decision=allow✅ 及格,可以 commit
decision=deny❌ 有舊契約被改壞了,把規則改回去或找工程師
skipped (pdf_missing)⚠️ 這台電腦沒有該案例的 PDF,跳過不算錯

5. 常見問題

Q:跑到一半可以關嗎?
可以。系統記得跑到哪,下次同一個指令會從斷點續跑,不會從頭來。

Q:同一份 PDF 想整個重跑?
把 data\projects\ 裡那個案子的資料夾刪掉,再跑一次即可。

Q:HTML 可以寄給別人嗎?
可以,單一檔案哪裡都能開。但「點頁碼跳 PDF」只有連同資料夾一起給(或對方自己跑)才有效。

Q:在另一台電腦用?

git clone -b learning-system https://github.com/acebd12345/wbsmaker.git
然後照 README 裝環境;真模型端點要自己建 wbs.local.toml 填(檔案不會跟著 git 走)。

Q:什麼東西絕對不能動?
tests\golden\contract_11108.pdf(黃金測試契約,唯讀)。
