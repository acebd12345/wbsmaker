# WBS 產生器 — 測試說明

把政府採購契約 PDF 丟進來,自動解析文件結構、用內網 LLM 抽取工作項,
產出**可追溯到頁碼**的 WBS(工作分解結構)。

## 安裝(離線,不需網路)

需求:Windows 10、Python 3.12(`py -3.12` 可啟動)。

1. 解壓縮本資料夾到任意位置(路徑避免中文與空格較保險)
2. 在資料夾內開 PowerShell,執行:

```powershell
.\install.ps1
```

所有相依套件都在 `wheels\` 裡,安裝過程完全離線。

## 使用

```powershell
# 一鍵全管線(以附的樣本契約為例,309 頁)
.\.venv\Scripts\wbs.exe auto sample\contract_11108.pdf

# 或自己的契約
.\.venv\Scripts\wbs.exe auto C:\path\to\你的契約.pdf
```

跑完後產物在 `data\projects\<專案名>\exports\`:

| 檔案 | 內容 |
|---|---|
| `wbs_v1_*.html` | **互動視覺化**(建議先開這個):WBS 樹狀圖 + 明細清單、類別過濾、頁碼回溯 |
| `wbs_v1_*.xlsx` | Excel 版 WBS |
| `wbs_v1_*.csv` / `.json` | 結構化資料 |
| `coverage.txt` | 頁面涵蓋率報告(幾頁正常/掃描/亂碼) |

## 兩種模式(wbs.toml 的 `mock` 開關)

| 模式 | 設定 | 速度 | 用途 |
|---|---|---|---|
| **真模型**(預設) | `mock = false` | 白天尖峰約 2–4 小時/份;離峰(清晨)約 15–30 分 | 正式測試,WBS 由內網 Gemma 模型語意分群 |
| Mock | `mock = true` | 約 1 分鐘 | 快速驗證安裝與管線,WBS 為規則式(照章節結構) |

> 內網 GPU 是共用的,白天呼叫一次要 50–75 秒,建議大份契約排離峰跑。
> 第一次測試可先改 `mock = true` 跑一遍確認環境沒問題,再改回 `false` 跑真模型。

## 系統會自動處理的髒資料

- **掃描頁**(整頁圖片):自動偵測、跳過,不會送進模型
- **亂碼頁**(字型內嵌錯誤,樣本契約的服務建議書 135 頁全是):自動偵測、排除
- **不該進 WBS 的文件**(投標須知、評選須知、招標公告、法規):自動分類為排除
- **跨頁表格**:自動合併、去除重複表頭
- 一份 PDF 裡的多份文件(契約本文/附件/需求規範書⋯)自動切分

## 測試時請幫忙留意

1. 開 `wbs_v1_*.html`,WBS 分支是否符合契約實際內容?
2. 點「付款」「驗收」類別過濾,項目的頁碼回溯是否正確?(對照原 PDF)
3. 「待歸入」區的項目是否其實很重要、應該進樹?(已知課題,回饋案例更好修)
4. 用你手上其他契約跑跑看——切分、分類錯誤的畫面截圖回報最有幫助

## 已知限制

- 掃描頁未做 OCR,其內容不會進 WBS
- 部分抽取出的工作項尚未被歸入樹中(顯示在「待歸入」區)
- 同一份 PDF 重跑會利用快取跳過已完成階段;要強制全部重跑,刪掉 `data\projects\<專案名>` 即可

## 問題回報

回報時請附上:契約 PDF(或頁數與問題頁碼)、`exports\coverage.txt`、問題畫面截圖。
